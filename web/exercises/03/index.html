Exercise 3: Somewhere fun to collect things

GAME 615 - American University

Description

Create a fun environment for a 3d model to move around in and collect objects! Below are the exercise requirements:

    • An "interesting" player should be controllable by the player
      The player should be able to collide with some object, and that object should disappear
    • That object should be a prefab
      The object should move or rotate (this will require creating a new script for the object)
      • Something about how the player moves should change when the object is collided with
      Use 3d models and colors to make the environment interesting in some way
      Feel free to make neat things happen with Rigidbodies and such
    Don't forget to check out the annotated sample project from class

    Getting the Sample Project Loaded into Your Project

    To integrate the project into another Unity project, download and import the .unitypackage file located at the "CODE" link below. In your own Unity project, go to Assets > Import Package > Custom Package...

    Once you do this, all of the assets from the sample project will be loaded into your project. Double click the "scene" file (the one with the Unity logo) to actually open up the sample game.

    Building your Project for the Web

    • Open the Build Settings (File > Build Settings...)
    • Click the Add Open Scenes button
    • Select "WebGL" and then click the "Switch Platform" button
    • Remember to modify your player settings (disable compression, turn off auto WebGL graphics or whatever if necessary.)
    • Click the "Build" button
    • When it asks you where you want to put your build, create a folder titled "play" in your main repository folder ("game615-spring2023-??"). Once you do this, it will take several minutes to build your game.

    Submission

    Turn in this assignment via your properly named GitHub repository. Each submission must be accompanied by a README.md file that contains your name, the name of any collaborators you had, and a link to a playable version of your game (i.e. a WebGL build) inside of a directory named "play" within your repository for the exericse. Also, on collaborations, only one member of the team should have the project in their repository and the rest of the team should be Contributors.

    Also, as part of your submission process, you must complete this form evaluation of your team members.
